## 1. 시스템 개요

### 1.1 목적
코그 인터내셔널의 일상 경영관리 업무를 디지털화하여 업무 효율성을 높이고, 법정의무사항 누락을 방지하며, 경영 현황을 실시간으로 파악할 수 있는 통합 관리 시스템.

### 1.2 기술 스택
| 항목 | 기술 |
|------|------|
| 프레임워크 | Next.js 16 (App Router) |
| 프론트엔드 | React 19, TypeScript 5 |
| 스타일링 | Tailwind CSS 4 |
| 데이터베이스 | Supabase (PostgreSQL) |
| 배포 | Vercel |
| 아이콘 | Lucide React |

### 1.3 사용자 역할
| 역할 | 설명 | 권한 범위 |
|------|------|-----------|
| admin | 시스템 관리자 | 모든 기능 접근, 사용자/부서 관리 |
| manager | 관리자급 | 데이터 CRUD, 직원 근태 관리 |
| staff/employee | 일반 직원 | 본인 업무일지 작성, 조회 |

### 1.4 데이터베이스 테이블 (18개)
- **핵심**: departments, users
- **업무**: work_logs, projects, project_members, project_milestones
- **교육**: education_schedules, education_targets, education_attendances
- **일정**: annual_schedules, special_workdays
- **HR**: hr_events, attendance_records
- **문서**: official_documents, document_histories
- **점검**: inspections, inspection_histories
- **시스템**: notifications

### 2.1 인증 및 로그인

#### 현재 구현 상태
| 기능 | 상태 | 비고 |
|------|------|------|
| 이름 + 비밀번호 로그인 | ✅ 구현 | POST /api/login |
| 샘플 모드 (데모) | ✅ 구현 | POST /api/sample-login, 24시간 유효 |
| 비밀번호 변경 | ✅ 구현 | POST /api/change-password, 최소 4자 |
| 쿠키 기반 세션 | ✅ 구현 | kog_user_id, 7일 유효, httpOnly |
| 로그아웃 | ✅ 구현 | 클라이언트에서 쿠키 삭제 |

#### 기능 요구사항
- **FR-AUTH-01**: 사용자는 이름과 비밀번호로 로그인할 수 있어야 한다
- **FR-AUTH-02**: 로그인 실패 시 명확한 오류 메시지 (미등록/비밀번호 오류 구분)
- **FR-AUTH-03**: 비활성화된 계정은 로그인 불가
- **FR-AUTH-04**: 샘플 모드로 전체 기능 체험 가능 (읽기 전용)
- **FR-AUTH-05**: 비밀번호 변경 시 현재 비밀번호 확인 필수

#### 알려진 문제 (Critical)
| ID | 문제 | 심각도 | 설명 |
|----|------|--------|------|
| BUG-AUTH-01 | 평문 비밀번호 저장 | 🔴 Critical | password_hash 필드에 평문 저장, bcrypt 등 해싱 필요 |
| BUG-AUTH-02 | 로그인 시도 제한 없음 | 🟡 Medium | 브루트포스 공격에 취약 |
| BUG-AUTH-03 | CSRF 보호 없음 | 🟡 Medium | 토큰 기반 CSRF 방어 필요 |

### 2.8 업무일지

#### 현재 구현 상태
| 기능 | 상태 | 비고 |
|------|------|------|
| 날짜별 업무일지 | ✅ 구현 | 이전/다음 날짜 이동, 캘린더 선택 |
| 업무 유형 3종 | ✅ 구현 | 정기업무/프로젝트/돌발업무 |
| 업무 등록 | ✅ 구현 | 유형, 제목, 설명 |
| 달성 토글 | ✅ 구현 | 원클릭, 미달성 사유 입력 |
| 삭제 (manager) | ✅ 구현 | 매니저 이상 |
| 근태 인라인 관리 | ✅ 구현 | 출근 상태 드롭다운 + 메모 |
| 마감 배너 (08:30 KST) | ✅ 구현 | 실시간 시계, 마감 임박/초과 표시 |
| 미작성자 표시 | ✅ 구현 | 사유별 구분 (연차/병가 vs 미작성) |
| 휴일/특근 처리 | ✅ 구현 | holidays.ts + special_workdays 테이블 |
| 유형/담당자 필터 | ✅ 구현 | 버튼 + 드롭다운 |
| 과거 날짜 읽기 전용 | ✅ 구현 | 당일만 등록/수정 가능 |

#### 기능 요구사항
- **FR-WL-01**: 직원은 당일 업무를 유형별로 등록
- **FR-WL-02**: 달성 여부를 토글하고, 미달성 시 사유 기록
