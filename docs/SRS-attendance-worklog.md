# 요구사항 분석서 (SRS) -- 근태 관리 / 업무일지 / 업무 리포트

**프로젝트:** KOG International 경영관리 지원 시스템 (KOG MSS)
**문서 버전:** 1.0
**작성일:** 2026-04-16
**작성자:** 시스템 개발팀

---

## 1. 개요

### 1.1 목적

본 문서는 KOG International 경영관리 지원 시스템의 세 가지 핵심 모듈에 대한 소프트웨어 요구사항을 정의한다.

1. **근태 관리 (Attendance Management)** -- 직원의 연차, 반차, 외근, 출장 등 근태 유형을 등록/승인/관리하는 모듈
2. **업무일지 (Work Log / Daily Report)** -- 직원이 당일 수행한 업무를 기록하고 달성 여부를 추적하는 모듈
3. **업무 리포트 (Work Report)** -- 당일/주간/월간 단위로 업무 현황을 집계하여 보고서를 생성하는 모듈

### 1.2 범위

- 근태 유형 8종의 등록, 승인/반려 워크플로우, 연차 잔여일수 추적
- 업무일지 CRUD, 달성 관리, 이전일 복사, 마감 관리
- 업무 리포트 생성(당일/주간/월간), KPI 요약, 부서별 필터, 인쇄/PDF 출력
- 샘플 모드를 통한 전체 기능 데모 체험 지원

### 1.3 용어 정의

| 용어 | 설명 |
|------|------|
| 근태 | 직원의 출근 상태 및 부재 유형 (연차, 반차, 외근, 출장, 재택, 병가 등) |
| 업무일지 | 직원이 매일 수행한 업무 내역을 기록하는 일일 보고 |
| 업무 리포트 | 업무일지 데이터를 집계하여 생성하는 정기 보고서 |
| 승인 워크플로우 | 직원 신청 -> 매니저 승인/반려의 1단계 결재 흐름 |
| 다일 등록 | 시작일~종료일 범위를 지정하여 연속된 근태를 한번에 등록하는 기능 |
| 연차 쿼터 | 연도별 총 연차 일수 및 사용 일수를 관리하는 할당량 |
| 달성률 | 등록된 업무 중 달성(완료) 처리된 업무의 비율 |
| KST | 한국 표준시 (Korea Standard Time, UTC+9) |
| 샘플 모드 | Supabase 연결 없이 하드코딩된 데모 데이터로 시스템을 체험하는 모드 |

| 배포 | Vercel | 자동 배포, Edge Functions |
| 인증 | 자체 구현 | bcrypt 비밀번호 해싱 |

| 역할 | 코드 | 권한 요약 |
|------|------|-----------|
| 시스템 관리자 | `admin` | 전체 시스템 관리, 모든 데이터 CRUD, 사용자/부서 관리, 근태 즉시 등록(승인 불필요) |

## 3. 기능 요구사항 (Functional Requirements)

### 3.1 근태 관리 (Attendance Management)

#### FR-ATT-001: 근태 유형 8종 관리

- 시스템은 다음 8종의 근태 유형을 지원해야 한다:
  1. 연차 (Annual Leave)
  2. 반차오전 (Half-day AM)
  3. 반차오후 (Half-day PM)
  4. 외근 (Off-site Work)
  5. 출장 (Business Trip)
  6. 재택 (Work from Home)
  7. 병가 (Sick Leave)
  8. 기타 (Other)
- 각 유형은 캘린더에서 고유한 색상 및 아이콘으로 표시된다.
- 반차(오전/오후)는 연차 0.5일로 계산된다.

#### FR-ATT-002: 승인 워크플로우

- **직원(staff)이 근태를 신청하면:**
  - `status='대기'`로 `attendance_records`에 INSERT 된다.
  - 직원은 대기 상태의 신청 건을 조회할 수 있다.

- **매니저(manager) 또는 관리자(admin)가 근태를 등록하면:**
  - `status='승인'`으로 즉시 INSERT 된다. (승인 워크플로우 생략)

- **승인 처리:**
  - 매니저/admin이 대기 상태 근태를 승인하면 `status='승인'`으로 변경된다.
  - `approver_id`에 승인자 ID, `approved_at`에 현재 시각이 자동 기록된다.

- **반려 처리:**
  - 매니저/admin이 대기 상태 근태를 반려하면 `status='반려'`로 변경된다.
  - `rejection_reason`(반려 사유) 입력이 필수이다.
  - `approver_id`, `approved_at`이 함께 기록된다.

#### FR-ATT-003: 다일(연속) 등록

- 시작일과 종료일을 지정하여 범위 내 모든 평일에 대해 근태를 일괄 등록할 수 있다.
- 공휴일 및 주말(토/일)은 자동으로 제외된다.
- 등록 전 확인 다이얼로그에 등록 대상 일수와 제외 일수가 표시된다.
- 각 날짜별로 개별 레코드가 생성된다 (upsert 처리, 동일 user_id+date 중복 방지).

#### FR-ATT-004: 연차 잔여일수 추적

- `leave_quotas` 테이블에서 연도별 총 연차 일수와 사용 연차 일수를 관리한다.
- 근태 등록 시 해당 연도의 잔여 연차가 표시된다.
- 연차 잔여일수 = 총 연차(`total_days`) - 사용 연차(`used_days`)
- 반차(오전/오후)는 0.5일로 계산된다.
- 잔여일수가 부족할 경우 경고 메시지를 표시한다 (등록은 가능).

#### FR-ATT-005: 직원 본인 조회

- staff 역할의 직원도 본인의 근태 기록을 캘린더 뷰에서 확인할 수 있다.
- 본인 기록만 필터링되어 표시된다.
- 대기/승인/반려 상태를 시각적으로 구분하여 표시한다.

#### FR-ATT-006: 캘린더 뷰

- 월별 그리드 형태의 캘린더를 제공한다.
- 각 날짜 셀에 해당일의 근태 건수 및 유형 아이콘이 표시된다.
- 날짜 클릭 시 해당일의 상세 근태 목록이 패널에 표시된다.
- 유형별 고유 색상 코드가 적용된다.
- 공휴일 및 주말은 시각적으로 구분된다.

#### FR-ATT-007: 월간 통계

- 선택된 월의 근태 유형별 건수가 집계되어 표시된다.
- 통계 항목: 유형별 건수, 전체 건수

#### FR-ATT-008: 부서/유형 필터링

- 부서(department) 드롭다운으로 특정 부서의 근태만 필터링할 수 있다.
- 근태 유형 드롭다운으로 특정 유형만 필터링할 수 있다.
- 필터는 캘린더 뷰와 통계에 동시 적용된다.

#### FR-ATT-009: 직원별 월간 요약

- 직원별 해당 월의 근태 사용 현황이 요약 표시된다.
- 연차 사용일수 / 잔여일수가 함께 표시된다.

#### FR-ATT-010: 근태 삭제

- 관리자(admin) 및 매니저(manager)만 근태 레코드를 삭제할 수 있다.
- 삭제 전 확인 다이얼로그가 표시된다.
- 삭제 시 관련 연차 사용일수가 자동 재계산된다.

### 3.2 업무일지 (Work Log / Daily Report)

#### FR-WL-001: 당일 업무 등록

- 직원은 당일 수행한 업무를 등록할 수 있다.
- 업무 유형 3종:
  1. 정기업무 (Regular)
  2. 프로젝트 (Project)
  3. 돌발업무 (Ad-hoc)
- 필수 입력 항목: 제목, 유형
- 선택 입력 항목: 설명, 예정 여부(is_planned)

#### FR-WL-002: 달성 토글 및 미달성 사유

- 각 업무 항목에 달성(achieved) 토글 버튼이 제공된다.
- 달성을 해제(미달성)할 경우 미달성 사유(note) 입력 영역이 활성화된다.
- 미달성 사유는 선택 입력이다.

#### FR-WL-003: 업무 편집

- 등록된 업무의 제목, 설명, 유형을 수정할 수 있다.
- 인라인 편집 모드로 전환되며, 저장/취소 버튼이 제공된다.
- 당일 업무만 직원이 편집 가능하다.

#### FR-WL-004: 이전일 업무 복사

- "어제 업무 복사" 버튼을 통해 전일 본인 업무를 조회할 수 있다.
- 복사할 항목을 체크박스로 개별 선택할 수 있다.
- 선택된 항목이 오늘 날짜로 복사된다 (achieved=false, note='' 초기화).
- 이미 동일 제목의 업무가 존재할 경우 중복 경고를 표시한다.

#### FR-WL-005: 08:30 KST 마감 배너

- 업무일지 페이지 상단에 실시간 시계와 마감 정보가 표시된다.
- 마감 시각: 당일 08:30 KST (다음 근무일 아침까지 전일 업무 작성)
- 상태별 표시:
  - 마감 전: 남은 시간 표시
  - 마감 임박 (30분 이내): 경고색 배너
  - 마감 초과: 초과 시간 표시, 적색 배너

#### FR-WL-006: 미작성자 추적

- 관리자/매니저는 당일 업무일지 미작성자 목록을 조회할 수 있다.
- 사유별 구분:
  - 근태 사유 (연차, 병가 등): 미작성 허용, 별도 표시
  - 미작성 (사유 없음): 강조 표시

#### FR-WL-007: 근태 인라인 관리

- 업무일지 화면에서 각 직원의 출근 상태를 드롭다운으로 관리할 수 있다.
- 출근 상태 유형: 정상출근, 연차, 반차, 외근, 출장, 재택, 병가, 기타
- 관리자/매니저만 다른 직원의 출근 상태를 변경할 수 있다.

#### FR-WL-008: 휴일/특근 처리

- `holidays.ts` 파일에 정의된 한국 공휴일 목록을 참조한다.
- `special_workdays` 설정으로 특별 근무일(임시공휴일의 대체 근무 등)을 지정할 수 있다.
- 휴일에는 업무일지 작성이 기본적으로 비활성화된다.
- 특근일에는 업무일지 작성이 활성화된다.

#### FR-WL-009: 과거 날짜 접근 제어

- **직원(staff):** 과거 날짜의 업무일지는 읽기 전용으로 표시된다.
- **관리자(admin)/매니저(manager):** 과거 날짜의 업무일지도 편집할 수 있다.

#### FR-WL-010: 유형/담당자 필터

- 업무 유형별(정기/프로젝트/돌발) 필터링이 가능하다.
- 담당자(직원)별 필터링이 가능하다.
- 필터는 조합 적용 가능하다.

### 3.3 업무 리포트 (Work Report)

#### FR-RPT-001: 당일/주간/월간 3종 보고서

- **당일 보고서:** 특정 날짜의 전 직원 업무 현황
- **주간 보고서:** 월요일~금요일 기준 주간 업무 현황
- **월간 보고서:** 해당 월 전체 업무 현황
- 탭 UI로 보고서 유형을 전환할 수 있다.
- 날짜 선택기로 조회 기간을 변경할 수 있다.

#### FR-RPT-002: KPI 요약

- 보고서 상단에 핵심 지표가 요약 표시된다:
  - 총 업무 건수
  - 달성 건수
  - 미달성 건수
  - 달성률 (%)

#### FR-RPT-003: 직책별 달성률 프로그레스 바

- 직책(position)별 달성률이 프로그레스 바 형태로 시각화된다.
- 달성률 수치가 바 옆에 표시된다.

#### FR-RPT-004: 부서별 필터

- URL 파라미터 `?dept=N`으로 부서별 필터링이 가능하다.
- 보고서 헤더에 부서 드롭다운이 제공된다.
- 부서 변경 시 URL 파라미터가 업데이트되어 공유/북마크가 가능하다.
- "전체 부서" 옵션으로 필터를 해제할 수 있다.

#### FR-RPT-005: 부서별 달성률 통계

- 부서별 달성률을 비교하는 통계 섹션이 보고서에 포함된다.
- 각 부서의 총 업무 건수, 달성 건수, 달성률이 표시된다.

#### FR-RPT-006: 미작성자 표시

- 당일 보고서에 한해 업무일지 미작성자 명단이 표시된다.
- 근태 사유(연차, 병가 등)와 미작성(사유 없음)을 구분하여 표시한다.

#### FR-RPT-007: 인쇄/PDF

- `window.print()` 기반의 인쇄 기능을 제공한다.
- 인쇄 시에만 결재란(작성자, 검토자, 승인자)이 표시된다.
- 인쇄 레이아웃은 A4 용지에 최적화된다.
- 화면 전용 UI 요소(필터, 버튼 등)는 인쇄 시 숨김 처리된다.

## 4. 비기능 요구사항 (Non-Functional Requirements)

### NFR-001: 응답 시간

- 모든 페이지의 초기 로드 시간은 2초 이내여야 한다.
- 데이터 조회/등록/수정 작업의 응답 시간은 1초 이내여야 한다.
- Supabase 쿼리 최적화 및 적절한 인덱싱을 적용한다.

### NFR-002: 반응형 디자인

- 모바일 (360px 이상), 태블릿 (768px 이상), 데스크톱 (1280px 이상) 해상도를 지원한다.
- Tailwind CSS의 반응형 유틸리티를 활용한다.
- 캘린더 뷰는 모바일에서 리스트 형태로 대체될 수 있다.

### NFR-003: 보안

- 비밀번호는 bcrypt 알고리즘으로 해싱하여 저장한다.
- 역할 기반 접근 제어(RBAC)를 적용한다:
  - admin: 전체 시스템 접근
  - manager: 소속 부서 데이터 관리
  - staff: 본인 데이터 조회/작성
- 인증되지 않은 사용자의 접근을 차단한다.

### NFR-004: 데이터 정합성

- `attendance_records`: `UNIQUE(user_id, date)` 제약으로 동일 날짜 중복 등록을 방지한다.
- `leave_quotas`: `UNIQUE(user_id, year)` 제약으로 연도별 쿼터 중복을 방지한다.
- 외래 키 제약으로 참조 무결성을 보장한다 (`user_id -> users`, `approver_id -> users`).

### NFR-005: 샘플 모드

- Supabase 연결 없이 하드코딩된 데모 데이터로 전체 기능을 체험할 수 있다.
- 샘플 모드에서는 데이터 변경(등록/수정/삭제)이 차단된다.
- 샘플 모드 활성화 시 화면 상단에 안내 배너가 표시된다.

### NFR-006: 한국어 인터페이스

- 모든 UI 텍스트, 라벨, 메시지는 한국어로 제공된다.
- 날짜 형식: YYYY-MM-DD (한국 표준)
- 요일 표기: 월, 화, 수, 목, 금, 토, 일

## 6. 제약사항

1. **공휴일 데이터:** 한국 공휴일 2025~2027년 데이터가 `holidays.ts` 파일에 하드코딩된다. 외부 API 연동은 지원하지 않는다.

2. **승인 워크플로우:** 1단계 승인만 지원한다 (직원 신청 -> 매니저/admin 승인). 다단계 결재(부장 -> 이사 -> 대표 등)는 지원하지 않는다.

3. **파일 첨부:** 근태 신청 및 업무일지에 파일 첨부 기능은 미지원한다.

4. **알림 기능:** 근태 승인/반려 시 이메일 또는 푸시 알림은 미지원한다. 시스템 내 조회로 확인한다.

5. **다국어:** 한국어 단일 언어만 지원한다.

6. **오프라인 모드:** 오프라인 환경에서의 데이터 입력 및 동기화는 지원하지 않는다.
