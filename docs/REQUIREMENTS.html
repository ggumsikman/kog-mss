1. 시스템 개요

코그 인터내셔널의 일상 경영관리 업무를 디지털화하여 업무 효율성을 높이고, 법정의무사항 누락을 방지하며, 경영 현황을 실시간으로 파악할 수 있는 통합 관리 시스템.

2.1 인증 및 로그인

Critical 이슈 1건
현재 구현 상태
기능상태비고
이름 + 비밀번호 로그인구현POST /api/login
샘플 모드 (데모)구현POST /api/sample-login, 24시간 유효
비밀번호 변경구현POST /api/change-password, 최소 4자
쿠키 기반 세션구현kog_user_id, 7일 유효, httpOnly
로그아웃구현클라이언트에서 쿠키 삭제
기능 요구사항
  • FR-AUTH-01 사용자는 이름과 비밀번호로 로그인할 수 있어야 한다
  • FR-AUTH-02 로그인 실패 시 명확한 오류 메시지 (미등록/비밀번호 오류 구분)
  • FR-AUTH-03 비활성화된 계정은 로그인 불가
  • FR-AUTH-04 샘플 모드로 전체 기능 체험 가능 (읽기 전용)
  • FR-AUTH-05 비밀번호 변경 시 현재 비밀번호 확인 필수
알려진 문제
ID문제심각도설명
BUG-AUTH-01평문 비밀번호 저장Criticalpassword_hash 필드에 평문 저장, bcrypt 등 해싱 필요
BUG-AUTH-02로그인 시도 제한 없음Medium브루트포스 공격에 취약
BUG-AUTH-03CSRF 보호 없음Medium토큰 기반 CSRF 방어 필요

6. 우선순위별 작업 목록

Phase 1: 시행 전 필수 수정 (4월 중)
  1. 비밀번호 해싱 적용 (bcrypt)
  2. 프로젝트 생성 시 owner_id/department_id를 로그인 사용자 기반으로
  3. 사용자 추가 시 부서 선택 기능
  4. 알림 읽음 처리 기능
  5. 무소속(department_id=null) 직원 처리
Phase 2: 시행 초기 필수 개선 (5월 1~2주)
  1. 직원 검색/필터
  2. 근태 연속 등록 (다일 선택)
  3. 프로젝트 팀원 관리 UI
  4. 프로젝트 기본 정보 수정
  5. 알림 클릭 시 해당 페이지 이동
  6. 사용자 정보 편집 (이름, 연락처 등)
Phase 3: 운영 안정화 (5월 3~4주)
  1. 공문서 파일 첨부
  2. 잔여 연차 관리
  3. 부서별 보고서 필터
  4. 점검/교육 알림 자동화
Phase 4: 고도화 (6월~)
  1. 실시간 알림 (SSE/WebSocket)
  2. Excel/CSV 내보내기
  3. 교육 이수 이력 관리
  4. RLS 보안 정책
  5. SMS/카카오 알림톡

7. 테스트 체크리스트

7.1 인증
  • 정상 로그인 (이름 + 비밀번호)
  • 잘못된 비밀번호 에러 메시지
  • 미등록 사용자 에러 메시지
  • 비활성 사용자 로그인 불가
  • 샘플 모드 진입 + 데모 데이터
  • 샘플 모드 데이터 변경 차단
